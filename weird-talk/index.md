## 1. 기본 원칙과 적용 범위

`각본대로`의 운영 주체인 릴리에스앤씨(공개 개발자명: 릴리게임즈, 이하 “운영자”)는 이용자의 개인정보를 중요하게 생각하며 개인정보 보호법 등 관련 법령을 준수합니다.

이 방침은 다음 배포 환경에 적용됩니다.

- Google Play에서 배포하며 게임 웹 자산을 APK/AAB에 포함한 Android WebView 앱
- 앱인토스에서 제공하는 게임 미니앱
- 개발 및 품질 검증을 위한 브라우저 실행본

Google Play Android 앱은 게임 화면, 대사와 이미지 등 웹 자산을 앱 패키지 안에서 읽습니다. 게임 데이터는 운영자 자체 서버에 저장하지 않지만, 이용 분석을 위한 Firebase JavaScript SDK와 광고 제공을 위한 Google Mobile Ads Next-Gen SDK(AdMob), Google User Messaging Platform SDK(UMP)의 HTTPS 통신에 `android.permission.INTERNET` 권한을 사용합니다.

앱인토스 버전은 배포 번들 안의 게임 자산을 사용하고, 광고 제공을 위해 앱인토스 통합 광고와 배너 광고 API를 사용합니다. 통합 광고는 실행 환경에 따라 Toss Ads 또는 Google AdMob 광고를 선택할 수 있습니다.

현재 서비스에는 회원가입, 로그인, 프로필, 채팅, 이용자 게시물, 인앱 결제, 클라우드 저장, 운영자 자체 서버 또는 원격 오류 수집 SDK가 없습니다. 모든 배포판은 Firebase용 Google Analytics를 사용합니다. Google, 토스, GitHub 또는 이메일 제공자가 자신의 서비스 제공을 위해 처리하는 정보에는 각 제공자의 개인정보처리방침과 약관이 함께 적용됩니다.

## 2. 개인정보 처리 요약

| 앱인토스 권한 | 별도 플랫폼 권한을 요청하지 않음 |
| 이용자가 직접 보내는 문의 | 지원 이메일로 문의할 때 발신 이메일 주소, 문의 내용과 자발적으로 첨부한 정보가 처리될 수 있음 |
| 정책 페이지 방문 | GitHub가 GitHub Pages 제공과 보안을 위해 IP 주소와 접속 로그를 독립적으로 처리할 수 있음 |
| 개인정보 판매 | 운영자가 개인정보를 판매하지 않음 |

## 3. 기기 안에 저장되는 정보

서비스는 게임 기능, 이용자 설정과 광고 노출 간격을 유지하기 위해 다음 정보를 이용자의 기기 또는 실행 환경에 저장할 수 있습니다.

| 저장 항목 | 목적 | 저장 위치 | 보유 기간 |
| --- | --- | --- | --- |
| 엔딩 도감 정보 | 이용자가 확인한 배드엔딩·해피엔딩과 도감 달성 상태 복원 | Android WebView, 앱인토스 또는 브라우저의 Local Storage (`gbd_dex_v1`) | 앱 데이터·사이트 데이터 삭제 또는 앱 제거 시까지 |
| 음소거 설정 | 효과음과 배경음의 켜짐·꺼짐 상태 복원 | 같은 Local Storage (`gbd_mute`) | 같은 기간 |
| 완료 횟수 | 다섯 번째 엔딩 전환마다 전면 광고 노출 시점을 계산 | 같은 Local Storage (`gbd_ad_completed_v1`) | 같은 기간 |
| 광고 처리 횟수 | 같은 엔딩 전환에서 전면 광고를 중복 요청하지 않도록 기록 | 같은 Local Storage (`gbd_ad_claimed_v1`) | 같은 기간 |
| 현재 플레이 상태 | 선택한 상황·배역·컷, 현재 선택지, 힌트와 이어하기 사용 여부 관리 | 실행 중인 앱 또는 웹 페이지의 메모리 | 앱·페이지 종료 시까지 |
| 정적 게임 자산 | 게임 화면, 대사와 이미지를 제공 | Google Play APK/AAB 또는 앱인토스 배포 번들 | 앱·미니앱 업데이트 또는 제거 시까지 |
| Google Analytics 클라이언트 ID | 고유 이용자와 세션 구분 및 이용 통계 산출 | 자사 쿠키 `_ga`와 Google Analytics가 사용하는 브라우저 저장소 | 앱·사이트 데이터 또는 쿠키 삭제 시 해당 기기에서 삭제. Google이 이미 처리한 정보는 Analytics 설정과 Google 정책에 따름 |

게임 Local Storage 키에는 이름, 이메일, 전화번호, 광고 ID 또는 플랫폼 계정 ID가 포함되지 않습니다. Google Analytics 클라이언트 ID는 게임 저장소와 별도로 관리되며 실제 이름이나 계정 ID를 커스텀 이벤트로 보내지 않습니다. 운영자는 로컬 게임 정보를 자체 서버에 저장하거나 이용자의 실제 신원과 결합하지 않습니다.

Google Play Android 게임 WebView는 Firebase용 Google Analytics의 자사 쿠키를 허용하고 제3자 쿠키는 허용하지 않습니다. 네이티브 광고 SDK의 저장과 네트워크 처리는 WebView 게임·Analytics 저장소와 분리되며 Google의 정책과 이용자 동의·광고 설정을 따릅니다. 앱인토스와 일반 브라우저에서 플랫폼 또는 브라우저가 자체 기능을 위해 관리하는 저장소와 캐시는 해당 플랫폼·브라우저의 설정과 정책을 따릅니다.

## 4. 운영자가 직접 수집하지 않는 정보

운영자는 서비스 화면을 통해 다음 정보를 직접 입력받거나 운영자 자체 서버에 저장하지 않습니다.

- 이름, 생년월일, 성별과 계정 비밀번호
- 이메일 주소와 전화번호
- 정확한 위치
- 주소록과 연락처
- 사진, 동영상과 파일
- 카메라와 마이크 입력
- 결제, 금융과 거래 정보
- 건강, 생체정보 등 민감정보
- 이용자의 실제 신원과 연결된 게임 진행 기록

다만 Firebase용 Google Analytics는 게임 이용 이벤트와 기본 이용자·세션·기기 정보를 처리하고, 광고 제공 과정에서는 Google Mobile Ads, Google UMP, Toss Ads 또는 앱인토스 광고 SDK가 IP 주소, 대략적인 위치, 광고·앱 상호작용, 진단 정보와 기기 식별자를 자동으로 처리할 수 있습니다. 이는 운영자가 게임 화면에서 이름이나 연락처를 직접 입력받거나 자체 서버에 저장하는 정보와 구별됩니다.

서비스에는 이용자가 개인정보를 입력하는 화면이 없습니다. 이용자가 지원 이메일에 자발적으로 정보를 포함하면 제6조에 따라 해당 문의 정보가 처리될 수 있습니다.

## 5. 광고, 분석과 네트워크 사용

### Google Play Android 버전

Google Play 버전은 Google Mobile Ads Next-Gen SDK를 통해 화면 하단 배너 광고와 전면 광고를 제공합니다. 전면 광고는 게임 결과 화면에서 다른 화면으로 이동하는 자연스러운 전환 중 완료 횟수 다섯 번마다 한 번 요청하며, 광고가 준비되지 않았거나 표시가 실패해도 게임 진행을 막지 않습니다.

Google의 공식 SDK 고지에 따르면 GMA Next-Gen SDK는 광고 제공, 광고 관련 분석과 부정 사용 방지를 위해 다음 정보를 자동으로 수집하고 공유할 수 있습니다.

- IP 주소: 기기의 대략적인 위치 추정에 사용될 수 있음
- 이용자 제품 상호작용: 앱 실행, 탭, 광고 동영상 조회 등
- 진단 정보: 앱·SDK 성능, 실행 시간, 중단률과 에너지 사용량 등
- 기기·계정 식별자: Android 광고 ID, 앱 세트 ID와 해당되는 경우 기기에 로그인된 계정 관련 식별자

Google은 해당 SDK 데이터가 전송 중 TLS로 암호화된다고 안내합니다. Android 광고 ID는 기기 설정에서 재설정하거나 삭제할 수 있고, 동의 상태 또는 제한된 광고 설정에 따라 광고 ID 전송이 제한될 수 있습니다.

앱은 UMP를 통해 실행할 때마다 동의 정보 갱신을 요청하고, `canRequestAds()`가 허용하는 경우에만 광고 SDK를 초기화하고 광고를 요청합니다. Google이 개인정보 선택 항목을 다시 제공하도록 요구하는 지역·상태에서는 게임 상단에 `광고 설정` 버튼을 표시합니다.

### 앱인토스 버전

앱인토스 버전은 `@apps-in-toss/web-framework`의 통합 전면 광고 API와 배너 광고 API를 사용합니다. 통합 광고는 토스 앱과 광고 그룹 설정에 따라 Toss Ads 또는 Google AdMob 네트워크를 선택할 수 있습니다. 광고 로드, 노출, 클릭, 측정, 부정 사용 방지와 정산을 위해 토스와 Google이 광고 관련 정보를 처리할 수 있으며, 해당 처리는 토스·Google의 정책과 플랫폼 동의·광고 설정을 따릅니다.

### Firebase용 Google Analytics

일반 웹, Google Play WebView와 앱인토스 배포판은 Firebase JavaScript SDK의 Google Analytics를 사용합니다. Google의 기본 구현 안내에 따르면 Google Analytics는 이용자 수, 세션 통계, IP 주소로 추정되는 대략적인 위치, 브라우저·기기 정보를 수집하고, 웹에서 고유 이용자와 세션을 구분하기 위해 자사 쿠키 `_ga`에 클라이언트 ID를 저장합니다.

서비스는 다음과 같은 게임 이용 이벤트를 추가로 전송합니다.

- 타이틀, 상황·배역 선택, 게임, 배드·해피 엔딩과 엔딩 도감 화면 진입
- 선택한 상황·배역의 고정 ID, 컷 번호, 선택 위치와 정답 여부
- 힌트와 무료 이어하기 사용, 배드·해피 엔딩 도달과 신규 도감 등록
- 엔딩 이후 이동 목적지와 다섯 번째 완료 시점의 전면 광고 처리 결과
- 배포판 구분(`web`, `google_play`, `apps_in_toss`)과 게임 버전

커스텀 이벤트에는 대사·엔딩 설명 등 자유 텍스트, 이름·이메일·전화번호, 플랫폼 계정 ID, Google Analytics 사용자 ID 또는 광고 식별자를 넣지 않습니다. 개발 로컬호스트, 자동화 브라우저와 스토어 이미지 캡처 모드에서는 Analytics 수집을 기본적으로 비활성화합니다.

### 공통 사항

배드엔딩의 `한 번 이어하기`는 광고 시청 보상이 아니라 루트당 한 번 제공되는 무료 기능입니다. 현재 서비스는 리워드 광고, Firebase Crashlytics, 운영자 자체 추적 서버 또는 별도 원격 오류 수집을 사용하지 않습니다. 운영자는 Analytics 콘솔에서 집계된 이용 현황과 이벤트를, 광고 콘솔에서 노출·클릭·광고 성과와 수익 등 집계 정보를 확인하지만 SDK의 원시 광고 식별자를 자체 서버에 저장하지 않습니다.

Google Play 앱의 게임 자산은 계속 `WebViewAssetLoader`의 앱 내부 로컬 오리진에서 제공되며 HTTP 평문 통신, WebView 제3자 쿠키, 위치 기능과 외부 파일 접근을 허용하지 않습니다. Google Analytics는 자사 `_ga` 쿠키와 HTTPS 통신을 사용하고, 광고 네트워크 통신은 네이티브 광고 SDK에서 HTTPS/TLS를 사용해 별도로 수행됩니다.
